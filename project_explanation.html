pngmodel — AI Logo Refiner

Fine-tuned Real-ESRGAN on logo data to upscale low-quality logos into high-res PNGs + clean SVGs. Trained entirely on a single RTX 3070 Ti (8 GB VRAM).

1. Goal

Take a low-resolution, artifact-ridden logo and produce:

  • A 4× upscaled, cleaned PNG (sharp edges, faithful colors, no hallucinated texture)
  • An SVG vector traced from the cleaned raster (via k-means color quantization + OpenCV contour detection)

3. The Model

Architecture: RRDBNet (Real-ESRGAN)

ComponentDetails
GeneratorRRDBNet — 23 Residual-in-Residual Dense Blocks, 64 feature channels, 32 growth channels, ×4 sub-pixel upscaling. ~16.7 million parameters. Input: 3-channel RGB. Output: 3-channel RGB at 4× resolution.
DiscriminatorU-Net with Spectral Normalization — ~4.38 million parameters. 4 down/up layers with skip connections. Judges per-pixel real-vs-fake, pushing the generator toward locally sharp detail.
Stage 1 (PSNR)RealESRNet: generator only, trained with L1 pixel loss. Builds a stable, structurally correct base. No discriminator → lower VRAM, faster iterations.
Stage 2 (GAN)RealESRGAN: generator + discriminator trained jointly with three losses: L1 pixel (weight 1.5), VGG-19 perceptual (weight 0.5), and GAN (weight 0.03). Adds crisp detail without hallucinating photographic texture.

Why fine-tune instead of training from scratch?

The official RealESRGAN_x4plus weights already know how to reverse real-world image degradation (downscaling, blur, JPEG noise). Starting from those weights means the model only needs to learn what makes a logo different from a photo — flat colors, hard edges, text, no noise. This saves enormous amounts of data and GPU time.

5. Before & After: The Dataset Journey

The biggest lesson from this project: the model can only output what it was shown during training. Here's how three dataset attempts taught that lesson.

Attempt 1 — FAILED: Simple Icons (monochrome)

3,443 icons → ~10,300 training images. Trained both stages. The result? Grayscale output with an embossed/metallic texture.

Root cause: Simple Icons are all single-color black silhouettes by design. The model faithfully learned "logos = grayscale" and the GAN hallucinated fake texture ("embossed metal") to add detail to flat shapes. It had never seen color, so it desaturated everything.

Lesson: Training data is the ceiling — not the model architecture.

Attempt 2 — REJECTED: Scraped colored JPGs

~2,000 colored logo JPGs. Rejected before training because the images had JPEG compression artifacts, collages, mockups, and template text. Bad ground truth = bad model output.

Attempt 3 — THE FIX: gilbarbara/logos (full-color vectors)

1,863 full-color brand SVGs → 1,860 clean PNGs → 5,580 training images (with multi-scale augmentation at 1.0, 0.75, 0.5, and 0.33×).

Verified the mix: ~75% colored, ~25% genuinely monochrome brand logos — a realistic, healthy distribution.

Loss weight tuning for logo art (vs. photo defaults)

Loss functionPhoto default Logo-tuned Why the change
L1 pixel 1.0 1.5 ↑ Trust the real colors more — logos have precise brand colors
VGG perceptual 1.0 0.5 ↓ Reduce texture invention — logos have flat regions, not fabric/grain
GAN (adversarial) 0.1 0.03 ↓ Stop the "embossed metal" hallucination on flat color shapes

Result: Color is preserved, edges are sharp, and the embossed texture artifact is gone. The model now upscales logos as logos.

7. End-to-End Pipeline

SVG brands → Render PNGs (CairoSVG) → Multi-scale augment → Stage 1: PSNR base (6h 21m) → Stage 2: GAN fine-tune (4h 37m) → Inference: 4× upscale → SVG trace (vtracer / OpenCV) → Gradio UI / CLI

Input: a single low-quality logo PNG — Output: a 4× high-res PNG + a scalable SVG vector.

8. Key Takeaway

It didn't fail because the AI was broken — it failed because it was fed black-and-white logos and asked to produce color. The fix wasn't a better model or hyperparameter tuning; the fix was better training data — clean color brand SVGs with properly tuned loss weights for flat logo art.
